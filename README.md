# leadgenv1

## Supabase setup

This project now sends lead submissions to Supabase from the contact form.

### 1) Create the table

Run this SQL in Supabase SQL Editor:

```sql
create table if not exists public.lead_submissions (
	id bigint generated by default as identity primary key,
	submitted_at timestamptz not null,
	name text not null,
	last_name text not null,
	company_name text not null,
	email text not null,
	country text not null,
	country_code text not null,
	country_flag text not null,
	industry text not null,
	industry_other text not null,
	organization_type text not null,
	job_function text not null,
	designers integer not null,
	cnc_machines integer not null,
	work_stations integer not null,
	projects_per_year integer not null,
	paper_prints_per_project integer not null,
	current_design_platform text not null,
	time_saved_percentage numeric not null,
	additional_projects_per_year numeric not null,
	paper_prints_eliminated integer not null,
	efficiency_boost_percentage numeric not null
);
```

### 2) Allow inserts

If RLS is enabled, create an insert policy for anonymous submissions.

```sql
alter table public.lead_submissions enable row level security;

create policy "Allow anonymous inserts"
on public.lead_submissions
for insert
to anon
with check (true);
```

### 3) Add project keys

In [script.js](script.js), set:

- `SUPABASE_URL` to your project URL
- `SUPABASE_ANON_KEY` to your anon/public key

By default, placeholders are used. Submission will be blocked until real values are added.
